# Software Design and Architecture

This repository contains implementations and exercises for various Software Design Patterns studied in the **Software Design and Architecture** course (Semester 6). These patterns provide proven solutions to common software design problems, helping to build scalable, maintainable, and robust applications.

## Design Patterns Included

The repository is organized by specific design patterns, categorized as follows:

### Creational Patterns
These design patterns provide various object creation mechanisms, which increase flexibility and reuse of existing code.
- **Factory Pattern** (`A_1_ Factory`)
- **Abstract Factory Pattern** (`Abstract-Factory`)
- **Builder Pattern** (`A_2_Builder`)
- **Singleton Pattern** (`A_3_Singleton`)

### Structural Patterns
These patterns explain how to assemble objects and classes into larger structures while keeping these structures flexible and efficient.
- **Adapter Pattern** (`A_4_Adapter`)
- **Bridge Pattern** (`A_5_Bridge`)
- **Composite Pattern** (`A_6_Composite`)
- **Facade Pattern** (`A_7_Facade`)

### Behavioral Patterns
These patterns are concerned with algorithms and the assignment of responsibilities between objects.
- **Observer Pattern** (`A_8_Observer`)
- **Strategy Pattern** (`A_9_Strategy`)

## Development
All implementation code is organized within its respective pattern folder. Feel free to explore each folder to see practical examples of how these patterns are applied.

> **Note:** The `0.Material` directory contains course reference materials, notes, and resources, which are ignored and not tracked by Git.
